## Overview

TaskDrawer presents a structured, numbered list of tasks generated by an AI
agent alongside a plan title and a one-line preview description. The drawer
collapses to a compact header strip and expands to reveal the task list,
showing up to 4.5 rows before scrolling.

The header shows the plan title, an optional preview description, a status
icon reflecting the current `planStatus`, and expand/collapse controls.

### When to use

- When an AI agent is executing a multi-step plan and users need visibility
  into its progress.
- When users need to be able to track, review, or dismiss a plan after
  completion.
- When a "generating" phase precedes execution and the drawer should
  auto-open once the plan is ready.

### When not to use

- For simple single-step operations that do not benefit from a visible task
  list.
- When the plan is not displayed adjacent to a text input — the slide-up
  entrance animation and rounded top corners are designed for placement
  directly above a prompt field.

## Formatting

### Anatomy

{/* TODO: add image — task-drawer-anatomy.png */}

1. **Root container** — `role="region"` landmark with rounded top corners;
   plays a 240 ms slide-up entrance animation on mount.
2. **Toggle button** — chevron icon button (ChevronDown / ChevronUp) that
   expands or collapses the drawer.
3. **Title** — single-line `$label-01` text; truncated with ellipsis on
   overflow.
4. **Preview** — secondary `$label-01` text shown only when expanded;
   animates in/out via CSS grid row transition.
5. **Status icon slot** — 16 × 16 px icon that reflects the current
   `planStatus` with a cross-fade animation on change.
6. **Body / task list** — scrollable region; hidden with `aria-hidden` +
   `inert` when collapsed.
7. **Task row** — numbered task item with a status icon; rows fade in via
   IntersectionObserver on first scroll into view.
8. **Mutation tag** — optional blue pill tag on `queued` tasks indicating
   `new`, `updated`, or `dropped`.

### Placement

{/* TODO: add image — task-drawer-placement.png */}

Place TaskDrawer immediately above the prompt input field. Set
`overflow: hidden` on the parent container so the 240 ms slide-up entrance
animation clips correctly.

The component renders at `width: 100%` and adapts to its container. Rounded
corners apply only to the top-left and top-right; the bottom edge is flat to
visually connect to the input below.

## Content

### Plan title

The `title` prop is the plan's human-readable name (e.g. `"Implementation
plan"`, `"Generating plan"`). It defaults to `"Updating plan"` when omitted
and is used as the accessible `aria-label` for the `role="region"` landmark —
use a meaningful, concise title.

### Preview description

The `preview` prop is a one-sentence summary shown below the title when the
drawer is expanded. Keep it under ~80 characters to avoid wrapping. It
defaults to a generic placeholder when omitted.

### Task labels

Each `PlanTask.label` is the task's human-readable description. Labels are
truncated with ellipsis when they overflow the row. During `generating`
status the label is replaced by a shimmer skeleton; provide a real label
before transitioning away from `generating`.

### Further guidance

For further content guidance, see Carbon's
[content guidelines](https://www.carbondesignsystem.com/guidelines/content/general).

## Behaviors

### Lifecycle phases

TaskDrawer maps `planStatus` to three lifecycle phases:

| Phase | `planStatus` values | Default drawer state |
| ---------- | ------------------------------------------ | -------------------- |
| Generating | `"generating"` | Collapsed |
| Executing | `"updating"` `"paused"` `"executing"` | User-controlled |
| Terminal | `"complete"` `"failed"` | User-controlled |

### Expand / collapse

{/* TODO: add image — task-drawer-collapsed.png */}

{/* TODO: add image — task-drawer-expanded.png */}

The drawer starts collapsed by default (`defaultExpanded={false}`). Pass
`defaultExpanded={true}` to open on mount.

Set `autoExpandOnGenerated={true}` to open automatically the first time
`planStatus` transitions away from `"generating"`. This has no effect if the
component mounts with any status other than `"generating"`. Use the
`onExpandedChange` callback to mirror expanded state in a parent component.

### Plan status icon

The header trailing area displays an icon that reflects the current plan
status:

| `planStatus` | Icon | Color token | Notes |
| ------------ | ------------ | ----------------- | ------------------------------------------ |
| `generating` | Renew (spin) | `$icon-secondary` | Spins continuously |
| `updating` | Renew (spin) | `$icon-secondary` | Shares key with `generating` — no re-trigger on transition |
| `paused` | Help | `$support-warning` | — |
| `executing` | Async | `$interactive` | — |
| `complete` | Checkmark | `$support-success` | Review and Dismiss actions appear |
| `failed` | Critical | `$support-error` | — |

### Complete state actions

When `planStatus === "complete"`:

- A **"Review"** ghost button appears in the header (only when `onReview` is
  provided).
- A **"Dismiss"** icon button (Close) appears at the far-right edge (only
  when `onDismiss` is provided).

### Task status visuals

Each task row is styled according to its `status`:

| `status` | Icon | Row tint | Label style |
| --------------- | ---------------- | --------------------------------------- | -------------- |
| `in_progress` | LoadingRing | None | Normal |
| `awaiting_input`| Help | `$notification-background-info` | Normal |
| `updating` | Renew (spin) | `$notification-background-info` | Normal |
| `error` | Warning | `$notification-background-warning` | Normal |
| `recovery` | Async | `$notification-background-warning` | Normal |
| `stopped` | IncompleteCancel | `$notification-background-error` | Normal |
| `failed` | Misuse | `$notification-background-error` | Normal |
| `complete` | CheckmarkOutline | `$notification-background-success` | Strikethrough |
| `queued` | Pending | None | Muted |
| `generating` | CircleDash | None | Skeleton shimmer |

### Mutation tags

Mutation tags are blue pill labels shown on `queued` tasks only, indicating
what changed during a plan revision:

- `new` — task was added in the latest revision.
- `updated` — task label or details changed.
- `dropped` — task was removed; also applies line-through to the label.

Tags animate in with a scale-up entrance and auto-clear when the task leaves
`queued` status. Use the `useTaskMutation` hook to manage them (see [Hooks](#hooks)).

## Hooks

Four companion hooks are exported from `@carbon-labs/react` to extend
TaskDrawer's capabilities.

### `useScrollToTask`

Manages a body ref and per-row ref registry so you can imperatively scroll to
any task.

```tsx
import { useScrollToTask } from '@carbon-labs/react';

const scroll = useScrollToTask(tasks);

<TaskDrawer
  tasks={tasks}
  bodyRef={scroll.bodyRef}
  registerItemRef={scroll.registerItemRef}
/>
```

Then call `scroll.scrollTo(target)` at any time:

| Target | Resolves to |
| ------------------ | -------------------------------------------------------------- |
| `'first'` | First task in the array |
| `'current'` | First task with `in_progress` / `awaiting_input` / `updating` |
| `'first-warning'` | First task with `error` / `recovery` / `failed` / `stopped` |
| `'last'` | Last task in the array |
| `string` (task id) | Task with that exact `id` |

### `useTaskMutation`

Manages mutation tags (`new` / `updated` / `dropped`) on `queued` tasks. Tags
auto-clear when a task's status changes away from `queued`.

```tsx
import { useTaskMutation } from '@carbon-labs/react';

const { mutations, mutateTasks } = useTaskMutation(tasks);

mutateTasks([
  { id: 'task-3', type: 'new' },
  { id: 'task-5', type: 'updated' },
  { id: 'task-6', type: 'dropped' },
]);

<TaskDrawer tasks={tasks} mutations={mutations} />
```

### `useTaskStatusUpdate`

Drives 120 ms two-phase (exit → enter) status transitions inside `TaskRow`.
Exported for consumers building custom task row renderers.

```tsx
import { useTaskStatusUpdate } from '@carbon-labs/react';

const { displayStatus, isTransitioning } = useTaskStatusUpdate(task.status);
```

### `usePlanEntryExit`

Manages the expand/collapse state and the `autoExpandOnGenerated` trigger.
Exported for advanced consumers who want to control expanded state outside
the full TaskDrawer wrapper.

```tsx
import { usePlanEntryExit } from '@carbon-labs/react';

const { expanded, handleToggle } = usePlanEntryExit({
  defaultExpanded: false,
  planStatus,
  autoExpandOnGenerated: true,
  onExpandedChange: (v) => console.log(v),
});
```
