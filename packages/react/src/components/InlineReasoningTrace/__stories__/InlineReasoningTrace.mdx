## Overview

The `InlineReasoningTrace` component displays AI reasoning steps in a clean,
hierarchical format. Each step can be expanded to show detailed content, with
support for processing states, task type icons, and governance indicators.

### Key Features

- **Expandable/Collapsible**: Accordion-style display with individual step
  expansion
- **Step Visibility Management**: Configurable initial visible steps with "Show
  more/less" functionality
- **Processing State**: Real-time processing indicators with FLIP and fade
  animation modes
- **Task Type Icons**: Support for both development and reasoning workflow icons
- **Governance Indicators**: Visual indicators for governed content and warnings
- **Smooth Animations**: GPU-accelerated transitions and animations

## Getting started

Install `@carbon-labs/react-inline-reasoning-trace` to use this component.

```bash
npm install @carbon-labs/react-inline-reasoning-trace
```

### Usage

```jsx
import { InlineReasoningTrace } from '@carbon-labs/react-inline-reasoning-trace';
import '@carbon-labs/react-inline-reasoning-trace/scss/inline-reasoning-trace.scss';

const steps = [
  {
    stepLabel: 'Initial Analysis',
    stepContent: 'Analyzing the input data...',
    taskType: 'analyzing',
  },
  {
    stepLabel: 'Pattern Recognition',
    stepContent: 'Identified three distinct patterns...',
    taskType: 'searching',
  },
  {
    stepLabel: 'Structured Output',
    stepContent: [
      {
        type: 'text',
        content: 'This item can come from a JSON-schema-compatible response.',
      },
      {
        type: 'code',
        language: 'ts',
        label: 'Example',
        code: 'const status = "ready";',
      },
    ],
    taskType: 'reasoning',
  },
  {
    stepLabel: 'React Slot',
    stepContent: (
      <span>
        App-authored JSX still works for <strong>custom rendering</strong>.
      </span>
    ),
  },
];

function MyComponent() {
  return (
    <InlineReasoningTrace
      triggerText="2 reasoning steps"
      steps={steps}
      openByDefault={false}
    />
  );
}
```

## Component API

### InlineReasoningTrace Props

| Prop                         | Type               | Default  | Description                                          |
| ---------------------------- | ------------------ | -------- | ---------------------------------------------------- |
| `steps`                      | `StepType[]`       | Required | Array of reasoning steps to display                  |
| `triggerText`                | `string`           | Required | Text to display in the collapsed trigger button      |
| `openByDefault`              | `boolean`          | `false`  | Whether to show expanded on load                     |
| `initialVisibleSteps`        | `number`           | `5`      | Number of steps to show initially before "Show more" |
| `showAllStepsByDefault`      | `boolean`          | `false`  | Whether to show all steps when opening the trace     |
| `allowStepCollapse`          | `boolean`          | `true`   | Whether to allow collapsing steps when all are shown |
| `isProcessing`               | `boolean`          | `false`  | Whether the trace is in processing state             |
| `currentProcessingStepIndex` | `number`           | `0`      | Index of the currently processing step               |
| `animationMode`              | `'fade' \| 'flip'` | `'flip'` | Animation mode for processing state transitions      |

### StepType

| Property      | Type                                   | Description                                                                           |
| ------------- | -------------------------------------- | ------------------------------------------------------------------------------------- |
| `stepLabel`   | `string`                               | Label for the reasoning step                                                          |
| `stepContent` | `React.ReactNode \| StepContentItem[]` | Content of the step (can be string, JSX, any React node, or structured content items) |
| `taskType`    | `TaskType`                             | Type of task (shows corresponding icon)                                               |
| `customIcon`  | `React.ReactNode`                      | Custom icon to display instead of task type icon                                      |
| `hasOverlay`  | `boolean`                              | Show warning overlay for this step                                                    |
| `isGoverned`  | `string \| boolean`                    | Governance level ("full", "partial", false)                                           |

### StepContentItem

Use structured step content when responses need to be serializable, schema
validated, or coerced into objects by tools such as LangChain. `text` and `code`
items are JSON-friendly. `custom` is a React-only escape hatch for app-authored
content and is not intended for LLM object output.

| Item   | Type                                                                | Description               |
| ------ | ------------------------------------------------------------------- | ------------------------- |
| Text   | `{ type: 'text'; content: string }`                                 | Plain text content        |
| Code   | `{ type: 'code'; code: string; language?: string; label?: string }` | Carbon code block content |
| Custom | `{ type: 'custom'; content: React.ReactNode }`                      | React-only custom content |

### Task Types

**Development Tasks:**

- `build`, `create`, `edit`, `update`, `run`, `test`, `deploy`, `misc`

**Reasoning Tasks:**

- `thinking`, `analyzing`, `searching`, `reviewing`, `validating`, `planning`,
  `executing`, `reasoning`

## Animation Modes

### FLIP Mode (Default)

Uses the FLIP (First, Last, Invert, Play) technique for buttery-smooth 60fps
GPU-accelerated animations. Provides smooth visual continuity as labels move
between positions.

### Fade Mode

Fade mode is still available as a fallback option, but Storybook examples use
the default FLIP transition except for the explicit fade and comparison stories.
